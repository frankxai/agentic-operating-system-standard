# Language Guide

The language should make the work feel serious, premium, and operationally credible.

## Preferred Terms

| Use | Avoid |
|---|---|
| Operating system | Prompt pack, prompt dump |
| Standard | Vibe, loose method |
| Module | Random repo |
| Workflow | One-off prompt |
| Loop | Habit, vague process |
| Ledger | Notes somewhere |
| Approval gate | Trust the agent |
| Evidence | Guess, vibe, assumption |
| Buyer | User avatar without buying context |
| First-win workflow | Feature list |

## Positioning Formula

Use:

```text
[Module] helps [buyer] turn [messy input] into [verified artifact] through [agents, skills, workflows, loops, and gates].
```

Examples:

- Creator OS helps creators turn ideas into publishable assets, offers, and launches through governed agent workflows.
- Business OS helps founders turn scattered priorities into revenue architecture, operating cadence, and decision ledgers.
- Investor OS helps investors and founders turn deal flow into diligence memos, risk registers, and portfolio support loops.

## Claim Rules

Do not claim:

- Assured revenue.
- Assured investment returns.
- Legal, financial, tax, or trademark certainty.
- Autonomous safety without gates.
- Verified facts without current evidence.

Use:

- "Triage."
- "Evidence-backed."
- "Human-approved."
- "Verification queue."
- "Decision ledger."
- "Risk-gated."
- "Public-safe."

## README Bar

A premium README should answer:

1. What is this?
2. Who is it for?
3. What does it produce?
4. Why is it different?
5. How does it work?
6. What is safe to do now?
7. What remains gated?
8. How does it connect to the ecosystem?
9. What is the first win?
10. How is it validated?
