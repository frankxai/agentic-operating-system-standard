# Adoption Playbook

## For A New Repo

1. Copy `templates/repo-readme.md` into the repo README.
2. Copy `templates/agents.md` into `AGENTS.md`.
3. Copy `templates/skills.md` into `SKILLS.md`.
4. Copy `templates/workflow.md` into `WORKFLOWS.md`.
5. Add module entry in a registry file.
6. Add validation.
7. Add first proof artifact.

## For An Existing Repo

1. Preserve existing instructions.
2. Add standard references without removing local repo rules.
3. Identify module type.
4. Define first-win workflow.
5. Add public/private gate.
6. Add proof artifact.
7. Add compliance level.

## For A Private Pro Repo

1. Keep sensitive workflows private.
2. Publish only public-safe starter docs.
3. Keep customer data out of examples.
4. Add explicit approval gates.
5. Add release and support path.

## Minimum Upgrade

If time is limited, add:

- `AGENTS.md`
- `SKILLS.md`
- `WORKFLOWS.md`
- `LOOPS.md`
- A single validation command
- A single proof artifact
