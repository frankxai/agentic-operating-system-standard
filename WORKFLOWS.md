# Workflows

Workflows turn intent into repeatable artifacts.

## Canonical Workflow: Module Bootstrap

1. Define buyer and first-win workflow.
2. Write module charter.
3. Define agents and skills.
4. Define loops and ledgers.
5. Add schemas and examples.
6. Add validation.
7. Publish public starter.
8. Keep pro implementation private when needed.

## Canonical Workflow: Agentic Creator OS

1. Intake creator objective.
2. Select content, product, media, launch, or audience lane.
3. Route to specialist agents.
4. Generate proof asset.
5. Package first offer.
6. Distribute.
7. Measure and improve.

## Canonical Workflow: Agentic Business OS

1. Establish business objective.
2. Map offer, buyer, channel, proof, and operations.
3. Route to founder, revenue, ops, customer, and risk agents.
4. Produce decision ledger.
5. Execute weekly cadence.
6. Review metrics and blockers.

## Canonical Workflow: Agentic Investor OS

1. Define thesis and mandate.
2. Capture deal or opportunity.
3. Build diligence queue.
4. Produce memo and risk register.
5. Route legal, financial, and compliance questions to human experts.
6. Track portfolio support loop.

This workflow is for organization and analysis. It is not financial, legal, tax, or investment advice.

## Canonical Workflow: Domain Intelligence

1. Capture candidate domains.
2. Score offline triage.
3. Verify availability, ownership, renewal, trademark, and market signals.
4. Route to buy, build, list, lease, hold, or pass.
5. Record decision ledger.
6. Revisit on review cadence.

## Canonical Workflow: Standard Adoption

1. Audit repo against compliance levels.
2. Add missing `AGENTS.md`, `SKILLS.md`, `WORKFLOWS.md`, and `LOOPS.md`.
3. Add module registry entry.
4. Add validation.
5. Add public/private gate.
6. Open issue or PR for remaining work.
