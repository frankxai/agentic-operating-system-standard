# Operating Loops

Loops are what make an agentic operating system improve instead of only execute once.

## Intent To Artifact Loop

```text
intent -> route -> agent contract -> artifact -> validation -> handoff -> ledger
```

Used by every module.

## Proof Loop

```text
claim -> evidence required -> source check -> risk gate -> publish or block
```

Used before public claims, reports, landing pages, investor materials, and customer deliverables.

## Revenue Loop

```text
buyer pain -> offer -> proof -> checkout -> onboarding -> outcome -> renewal or upsell
```

Used by Business OS, Creator OS, Domain Intelligence, and productized services.

## Creator Loop

```text
source idea -> angle -> asset -> distribution -> feedback -> product -> audience memory
```

Used by Creator OS and media modules.

## Investor Loop

```text
thesis -> deal flow -> diligence -> memo -> decision -> portfolio support -> learning
```

Used by Investor OS. Human review is required for investment, legal, tax, and regulated decisions.

## Business Command Loop

```text
objective -> priority stack -> execution wave -> metrics -> blockers -> decision ledger
```

Used by Business OS and founder command.

## Repo Quality Loop

```text
standard -> repo files -> validation -> CI -> release note -> adoption feedback
```

Used by standard repos and module repos.

## Memory Loop

```text
event -> classify -> store -> retrieve -> apply -> evaluate -> prune
```

Used by SIS-aligned systems. Memory should preserve useful context without leaking private data.
